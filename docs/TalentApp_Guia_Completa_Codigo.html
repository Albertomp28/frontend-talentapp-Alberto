1. Vision General del Proyecto

TalentApp es una plataforma web moderna de reclutamiento construida con React. Permite a los reclutadores gestionar vacantes laborales, analizar CVs con simulacion de inteligencia artificial y administrar un pipeline de contratacion visual tipo Kanban.

Funcionalidades Principales:
  • Autenticacion: Sistema de login simulado con tokens en localStorage
  • Tablero Kanban: Pipeline visual de reclutamiento con drag-and-drop nativo
  • Gestion de Vacantes: Crear, editar, pausar y eliminar vacantes con formulario de 3 pasos
  • Analisis de CV: Subida de CV con simulacion de analisis IA (scores aleatorios)
  • Pool de Candidatos: Base de candidatos previamente evaluados con filtros

3.2 PrivateRoute.jsx - Proteccion de Rutas

Este componente actua como un "guardian". Envuelve las rutas que requieren autenticacion y verifica que exista un token valido antes de permitir el acceso.

src/router/PrivateRoute.jsx 
import { Navigate, useLocation } from 'react-router-dom';

const PrivateRoute = ({ children }) => {
  const location = useLocation();  // Obtiene la URL actual

  // Verifica si hay token de autenticacion
  const getToken = () => localStorage.getItem('auth_token');
  const isAuthenticated = !!getToken();  // Convierte a booleano

  if (!isAuthenticated) {
    // Redirige a /login guardando la URL original
    // Asi despues del login puede volver a donde estaba
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return children;  // Si esta autenticado, muestra el contenido
};
Nota importante: Actualmente la autenticacion es simulada. Cualquier email/password funciona porque el backend aun no esta implementado. El token se genera localmente con Date.now(). En produccion, esto se reemplazaria por llamadas a una API real con JWT.

Drag and Drop (Arrastrar y Soltar):

Implementado con la API nativa de HTML5 Drag and Drop (sin librerias externas). El flujo es:

  1. onDragStart - Guarda el candidato arrastrado en el estado
  2. onDragOver - Resalta la columna destino (borde azul)
  3. onDrop - Actualiza la columna del candidato en el estado
  4. onDragEnd - Limpia el estado de arrastre

Carga y Analisis de CV (CVUploadModal):

Modal de 3 pasos para simular el analisis de un CV con IA:

  1. Paso 1: Subir archivo + llenar nombre, email, telefono, vacante
  2. Paso 2: Animacion de "analizando" (2.5 segundos)
  3. Paso 3: Muestra resultados con scores aleatorios (experiencia, habilidades, educacion, cultura) y habilidades detectadas
Simulacion: El analisis de CV es simulado. Los scores se generan aleatoriamente entre 70-100. Las habilidades se seleccionan al azar de un pool predefinido. En produccion, esto se conectaria a un LLM real.

8. Hooks Personalizados

Los Custom Hooks son el patron central de TalentApp para separar la logica del estado de la presentacion visual. Cada pagina compleja tiene su propio hook.

HookPaginaQue maneja
useLogin Login Estado del formulario (email, password, loading, error). Validacion y llamada a authService.

useKanbanBoard KanbanBoard Candidatos, drag-and-drop, filtros (vacante, score minimo), modal de detalle, subida de CV, analisis simulado.

useVacanteForm CrearVacante Formulario de 3 pasos, validacion por paso, modo edicion, habilidades/beneficios, submit a localStorage, modal de exito.
Patron comun en los hooks:
  1. Definen estado con useState
  2. Cargan datos iniciales con useEffect
  3. Calculan valores derivados con useMemo
  4. Memoizan funciones con useCallback
  5. Retornan un objeto con estado, setters y acciones

11. Despliegue (Docker y Nginx)

Dockerfile - Build Multi-etapa

El Dockerfile usa 3 etapas para optimizar el tamano de la imagen final:

EtapaImagen BaseProposito
developmentnode:20-alpineServidor de desarrollo con hot-reload (Vite)
buildnode:20-alpineCompila la app para produccion (npm run build)
productionnginx:alpineSirve los archivos estaticos compilados con Nginx

docker-compose.yml

Define dos servicios:

  • dev: Para desarrollo local. Monta el codigo como volumen para hot-reload. Puerto 5173.
  • prod: Para produccion. Usa Nginx. Puerto 80.

nginx.conf

Configuracion del servidor web para produccion:

  • SPA routing: try_files $uri $uri/ /index.html - Todas las rutas redirigen a index.html (necesario para React Router)
  • Compresion gzip: Reduce el tamano de transferencia
  • Cache de assets: JS, CSS e imagenes se cachean por 1 ano (immutable)
  • Headers de seguridad: X-Frame-Options y X-Content-Type-Options
